# 星际穿越 · 随机抽奖

基于 Three.js 的沉浸式随机抽奖网页，模拟星球大战超空间跳跃效果，穿越星际随机降落在目标星球上揭晓结果。

## 效果预览

- 超空间跳跃动画（星星拉伸光束、星云背景）
- 1~3 颗星球同时飞越，随机大小和位置
- 最终选定星球着陆，大气层菲涅尔辉光
- Bloom 后期处理、扫描线、暗角

## 快捷键

| 按键 | 功能 |
|------|------|
| `空格` / `S` | 开始抽奖 / 跳过动画直接出结果 / 揭示下一位中奖者 / 全部揭示后重新开始 |
| `M` | 开关音效 |
| `,` | 打开设置 |
| `.` | 关闭设置 |

> 输入框聚焦时快捷键自动禁用。

## 设置面板

右上角 ⚙ 按钮打开，两个 Tab：

### Tab 1：数据源

#### 预设数据

一键加载内置数据集，标题自动切换：

| 预设 | 标题 | 条目数 |
|------|------|--------|
| 🍽 吃什么 | 吃什么😋 | 100 |
| 🧋 喝什么 | 喝什么🧋 | 50 |
| 🍰 甜品 | 吃什么甜品🍰 | 40 |
| 🍎 水果 | 吃什么水果🍎 | 40 |
| 🍿 零食 | 吃什么零食🍿 | 40 |

#### 数据格式

支持两种输入格式，自动识别：

**TXT 格式（一行一个）：**
```
张三
李四
王五
```

**CSV 格式（id,name,emoji 或 imgUrl）：**
```
1,火锅,🍲
2,烧烤,🍖
3,奶茶,🧋
```

**CSV 格式（id,name,imgUrl）— 显示图片：**
```
1,张三,https://example.com/zhangsan.jpg
2,李四,https://example.com/lisi.png
3,王五,https://example.com/wangwu.jpg
```

- 第三列自动检测：以 `http` 开头或含文件扩展名（`.jpg` `.png` 等）→ 识别为图片 URL
- 否则作为 emoji 显示
- 缺省第三列时默认显示 ⭐

#### 数据操作

- **粘贴**：直接在文本框粘贴数据
- **拖拽上传**：拖入 `.txt` 或 `.csv` 文件
- **点击上传**：点击文本框区域选择文件
- **载入数据**：点击按钮生效
- **恢复默认**：回到内置 8 颗星球数据，标题恢复「星 际 穿 越」

### Tab 2：选项

- **标题文字**：自定义页面标题，实时生效
- **音效**：开关引擎启动音 + 中奖提示音
- **选中个数**：`−` / `+` 调整，上限为当前数据条数

## 多人中奖流程

设置选中个数 > 1 时：

1. 按空格 / 点击「启 动 引 擎」→ 穿越动画 → 揭晓第 1 位
2. 按空格 / 点击「下一位 ⏭」→ **新一轮穿越动画** → 揭晓第 2 位
3. 重复直到全部揭晓
4. 按空格 / 点击「再次穿越」→ 重新开始

每位中奖者都有独立的完整穿越动画，非一次性全部展示。

## 中奖结果显示

- 有 imgUrl → 圆形头像（80×80，object-fit: cover）
- 无 imgUrl → emoji 图标
- 下方显示 name

## 图片缓存

载入数据或选择预设时，自动预加载所有 imgUrl 到内存缓存（`Map<url, Image>`）：

- 载入自定义数据 → 立即预加载所有图片
- 选择预设 → 立即预加载该预设所有图片
- 中奖结果展示时直接从缓存读取，无网络延迟
- 图片加载失败自动从缓存移除，结果卡片降级显示 emoji

## 音效系统

### BGM

- 设置面板中可上传/选择 BGM（支持 mp3/ogg/wav）
- BGM 使用 `Blob URL` 加载，避免重复网络请求
- 循环播放，音量可调
- 关闭音效时 BGM 同步暂停，开启时恢复

### 事件音效

不同事件触发不同音效，均使用 `Blob` + `URL.createObjectURL` 加载：

| 事件 | 音效描述 |
|------|----------|
| 引擎启动 | 引擎轰鸣渐强 → 超空间跳跃 |
| 星球飞越 | 嗖嗖掠过声 |
| 选中揭晓 | 叮咚提示音 |
| 下一位 | 短促确认音 |
| 重置 | 低沉消退音 |

- 音效文件预加载为 `ArrayBuffer` → 转 `Blob` → `createObjectURL`
- 设置面板中可为每个事件单独上传自定义音效
- 内置合成默认音效（Web Audio API），无需外部文件即可运行

## 技术栈

- Three.js（ES Module）
- TypeScript
- Bun（构建 + 开发服务器）
- 纯 HTML/CSS/JS → TS 重写
- 程序化生成星球纹理（Canvas），无需外部图片
- Web Audio API 合成默认音效
- Bloom 后期处理（UnrealBloomPass）

## 内存优化

- 所有星球复用同一个 `SphereGeometry`
- 星星粒子使用单个 `BufferGeometry` + `ShaderMaterial`，3000 颗复用一套缓冲区
- 通过 `visible` 控制显隐，避免反复创建/销毁
- 纹理在 `rebuildPlanets` 时按需生成
- 图片预加载后缓存为 `Image` 对象，结果展示时直接复用，避免重复请求
